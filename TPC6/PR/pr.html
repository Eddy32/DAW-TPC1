Abstract

O quinto trabalho de caso era composto por criar um website relativo ao registo de tarefas e visualização das mesmas.

Este teria de ser capaz de responder a pedidos GET e POST e estar devidamente preparado para o caso de receber como argumento algo "inesperado", evitando assim um cenário de erro por parte do servidor.

Graficamente o servidor é composto por uma página inicial que contém uma form de submissão de tarefas e abaixo uma tabela com todas as que foram criadas. Adicionalmente foi criada uma página de erro, para processar os pedidos do utilizador a que servidor não tinha resposta.
